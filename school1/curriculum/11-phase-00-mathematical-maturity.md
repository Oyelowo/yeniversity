# Phase 00 - Mathematical Maturity

This phase is the real beginning.

Before algebra, calculus, physics, or engineering become durable, you need mathematical discipline: precise statements, careful definitions, comfort with notation, and the habit of proving or disproving claims rather than guessing.

## Why This Phase Exists

Many people think they are weak at mathematics when the real issue is earlier:

1. They read symbols loosely.
2. They confuse examples with definitions.
3. They use intuition without checking conditions.
4. They do not distinguish a statement from its converse.
5. They have not trained recall and exactness.

Fix that first and everything after it becomes more stable.

## Lessons

1. `Lesson 00`: How to study, write, and check mathematics
2. `Lesson 01`: Propositions, truth values, and logical connectives
3. `Lesson 02`: Implication, equivalence, converse, inverse, contrapositive
4. `Lesson 03`: Sets and set operations
5. `Lesson 04`: Functions, mappings, and composition
6. `Lesson 05`: Quantifiers and statement parsing
7. `Lesson 06`: Proof patterns and counterexamples

## Phase Outputs

By the end of this phase, you should be able to:

1. Parse symbolic statements precisely.
2. Build truth tables correctly.
3. Detect invalid reasoning patterns.
4. Use set and function notation without sloppiness.
5. Read and write quantified statements.
6. Produce a direct proof, contrapositive proof, and counterexample.

## How To Work Through A Lesson

For each lesson:

1. Read the note carefully.
2. Rewrite the definitions from memory.
3. Do the exercises by hand first.
4. Check your own errors explicitly.
5. If there is a Rust companion, implement it.

## Exit Test

Do not leave this phase until you can explain the difference between:

1. `p -> q` and `q -> p`
2. a set and an element
3. a function's domain and codomain
4. `for all` and `there exists`
5. a true statement and a valid argument
