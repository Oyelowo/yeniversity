# Master Roadmap

This curriculum is organized to support the long-term goal: understand, design, simulate, and build advanced engineered systems from first principles.

## North Star

The target is not perfect knowledge of every field. The target is deep, composable mastery of the foundations so you can derive, model, and build across domains.

Core capacities:

1. Mathematical fluency
2. Physical intuition grounded in equations
3. Computational modeling in Rust
4. Electronics and embedded systems competence
5. Control and robotics competence
6. AI / ML from math and systems, not API usage
7. Systems engineering thinking for large devices and infrastructure

## Sequence

The order matters.

1. Mathematics foundations
2. Classical physics foundations
3. Chemistry and materials for engineering
4. Electronics, computing, and embedded systems
5. Signals, systems, control, and robotics
6. AI / ML and intelligent systems
7. Large-scale engineering systems and infrastructure

## Phases

### Phase 1: Re-entry and Fluency

Duration: 3 to 6 months

Deliverables:

1. Strong algebra and trigonometry fluency
2. Calculus I completed
3. Mechanics basics refreshed
4. Rust scientific programming basics in place
5. 3 to 5 small simulation programs

### Phase 2: Core Analytical Foundation

Duration: 6 to 12 months

Deliverables:

1. Calculus II
2. Linear algebra
3. Differential equations
4. Probability and statistics basics
5. Electricity and magnetism basics
6. Circuit analysis fundamentals

### Phase 3: Engineering Base Layer

Duration: 6 to 12 months

Deliverables:

1. Analog and digital electronics
2. Signals and systems
3. Computer architecture fundamentals
4. Embedded Rust workflow
5. At least 2 hardware projects and 5 numerical simulations

### Phase 4: Integrated Systems

Duration: 6 to 12 months

Deliverables:

1. Control theory basics
2. Estimation and sensor fusion
3. Robotics basics
4. AI / ML foundations
5. One integrated robot or autonomous system project

### Phase 5: Domain Specialization

Duration: ongoing

Choose from:

1. Drones and aerospace
2. Vehicles and autonomy
3. Cameras, vision, and sensors
4. Chips, digital systems, and fabrication fundamentals
5. Infrastructure and industrial systems

## Daily Study Pattern

Use this pattern for each session:

1. `15 min`: recall yesterday from memory
2. `45 to 90 min`: new theory
3. `45 to 90 min`: exercises
4. `30 to 60 min`: Rust simulation, coding, or project work
5. `10 min`: summary and error log

## Progress Gates

Do not advance a topic until you can:

1. Reproduce the core derivation
2. Solve standard exercises without peeking
3. Implement a working numerical model in Rust
4. Explain where the model breaks down
5. Connect the topic to a real device or system

## Domain Outcomes

If you execute this well, you should eventually be able to:

1. Design and simulate mechanical, electrical, and control systems
2. Build drones, robots, instruments, and embedded devices
3. Understand the architecture of cars, cameras, smartphones, and infrastructure systems
4. Build simpler working versions of many advanced technologies
5. Reason honestly about what is solo-feasible versus industrial-scale
