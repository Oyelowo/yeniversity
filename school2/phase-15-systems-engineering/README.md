# Phase 15 — Systems Engineering

**Duration:** 2–3 months  
**Prerequisites:** All prior phases (this phase synthesises and integrates everything)  
**Best time:** After Phases 06–11; can be studied in parallel with later phases

---

## Why This Phase Exists

You now know how to build individual subsystems. Systems engineering is the discipline of making
many interacting subsystems work *together* reliably as a whole — at scale — under real-world
constraints of mass, power, cost, risk, schedule, and uncertainty. This is how NASA builds a
Mars rover, how Boeing builds an aircraft, how TSMC builds a fab, and how SpaceX builds
Starship. It is the difference between "I can build the parts" and "I can build the thing."
The methods here — requirements decomposition, interface definitions, reliability modelling,
FMEA, systems simulation — apply to every complex artefact you will ever create.

## Topics

### 1. The Systems Engineering Process

#### What Systems Engineering Is
- Definition: the interdisciplinary approach to enabling the realisation of successful systems
- Why it exists: managing complexity; emergent properties; interface failures; integration risks
- Historical context: V-2, Apollo, Challenger, Columbia — systems failures and what they teach
- Concurrent engineering vs. sequential (waterfall) development

#### System Lifecycle
- Phases: concept → preliminary design → critical design → integration → test → operations
- Decision gates (PDR, CDR, MRR, TRR, ORR — NASA terminology)
- Technical readiness levels (TRL): 1–9 scale; use in risk assessment

### 2. Requirements

#### Requirements Development
- Need → stakeholder requirements → system requirements → subsystem requirements (decomposition)
- Characteristics of good requirements: unambiguous, verifiable, traceable, necessary, consistent
- Functional requirements: what the system must *do*
- Non-functional (performance/constraint) requirements: how well, within what limits
- Derived requirements: generated by design decisions; must trace back to parent requirements

#### Requirements Verification
- Four methods of verification: analysis (mathematical proof), inspection (visual check), demonstration (observe function), test (measurement)
- Verification and Validation (V&V): verification = built it right; validation = built the right thing
- Requirements traceability matrix (RTM): maps each requirement to its verification method and status

### 3. Architecture and Design

#### Functional Architecture
- Functional decomposition: break system function into sub-functions; functional block diagrams
- Functional flow block diagrams (FFBDs); structured analysis
- State machine descriptions for modes and operating states

#### Physical Architecture
- Allocation of functions to physical components; mapping F-arch to P-arch
- System block diagram; data flow diagrams; control flow
- Trade studies: choosing between alternative architectures; figures of merit; weighted scoring

#### Interface Management
- Interface Control Documents (ICDs): define mechanical, electrical, data, thermal interfaces
- Interface types: mechanical (bolts, pins, mating surfaces), electrical (voltage/current/impedance/pinout), data (protocol, format, rate, latency), thermal (heat flux, temperature limits)
- Interface verification: how do you confirm the interface works before integration

### 4. Budgets and Allocations

#### Mass Budget
- System-level mass allocation to each subsystem; margin philosophy
- Growth allowance (GA) and mass margin; mass tracking through development
- Unit mass + harness + structure + margin = system mass

#### Power Budget
- Power modes: eclipse/daylight (spacecraft); idle/active/peak (general)
- Worst-case power consumption analysis; power margin
- Battery sizing: capacity (Wh); depth of discharge; cycle life; state of charge management

#### Link Budget (Communications)
- Transmit power, antenna gain, free-space path loss, noise figure, required $E_b/N_0$
- Received signal strength; $C/N_0$; link margin
- Shannon capacity theorem: $C = B \log_2(1 + SNR)$; limits on communication rate

#### Thermal Budget
- Heat sources: electronics dissipation; solar/IR inputs; Earth albedo
- Heat rejection: radiators; conductive paths; heat pipes; thermoelectric coolers
- Thermal margin: operating temperature range for each component
- Simple thermal model: lumped parameter network; conductances and capacitances; steady-state temperature

#### Error Budget (Pointing / Navigation Accuracy)
- Root sum square (RSS) of individual error sources: $\sigma_{total} = \sqrt{\sigma_1^2 + \sigma_2^2 + \cdots}$
- Allocation of error to sensors, algorithms, actuation
- Margin in error budget

### 5. Reliability and Risk

#### Reliability Theory
- Reliability function $R(t) = P(T > t)$; failure rate $\lambda(t)$; bath-tub curve
- Exponential distribution: constant failure rate; $R(t) = e^{-\lambda t}$; MTTF = $1/\lambda$
- Weibull distribution: $R(t) = e^{-(t/\eta)^\beta}$; shape parameter $\beta$; infant mortality, random, wearout
- Series systems: $R_s = \prod R_i$; reliability drops with number of components
- Parallel (redundant) systems: $R_p = 1 - \prod (1 - R_i)$; redundancy improves reliability
- k-of-n voting systems
- Reliability block diagrams (RBD)

#### Failure Mode and Effects Analysis (FMEA)
- For each component: identify failure modes → effects at next level → effects on system
- Severity, occurrence, detectability ratings; Risk Priority Number (RPN) = S × O × D
- Critical items list (CIL): failures with catastrophic/critical effects
- Mitigation actions: redesign, add redundancy, add detection, change materials

#### Fault Tree Analysis (FTA)
- Top-down: start from undesired top event; decompose with AND / OR gates
- Minimal cut sets: smallest sets of basic events that cause top event
- Quantitative FTA: combine failure probabilities through gates to compute top-event probability
- Comparison to FMEA: FTA is top-down; FMEA is bottom-up; both needed

#### Risk Management
- Risk register: identify, assess (likelihood × consequence), mitigate, track
- Risk matrix: 5×5 likelihood-consequence grid
- Residual risk acceptance criteria; risk reviews at decision gates

### 6. Integration, Test, and Verification

#### Test Planning
- Unit test: test individual component or assembly
- Subsystem test: test an assembled subsystem (flight-like or engineering model)
- System integration test (SIT): integrate all subsystems; verify interfaces
- Environmental test: vibration, thermal vacuum, EMC/EMI testing

#### Qualification and Acceptance
- Qualification testing (engineering model): demonstrate design survives worst-case environments with margin
- Acceptance testing (flight hardware): verify unit-level workmanship and function; usually less severe than qual
- Test-like-you-fly philosophy

### 7. Spacecraft Systems (Case Study — Synthesises Everything)

A spacecraft is the most complete systems engineering challenge: every discipline applies.

#### Power System
- Solar arrays: power vs. sun angle; end-of-life degradation; sizing for eclipse power
- Batteries: Li-ion; depth of discharge trade-off; charge control electronics
- Power conditioning and distribution unit (PCDU); regulated vs. unregulated bus

#### Attitude Determination and Control System (ADCS)
- Sensors: star tracker, sun sensor, magnetometer, rate gyroscope, GPS
- Actuators: reaction wheels (momentum exchange), magnetorquers (dump momentum), thrusters
- Control modes: detumble → coarse sun → fine pointing → manoeuvre
- Connects directly to Phase 05 (control theory) and Phase 06 (estimation/Kalman filter)

#### Thermal Control System
- Passive: surface coatings (emissivity/absorptivity ratio); multi-layer insulation (MLI); radiators
- Active: heaters; heat pipes; loop heat pipes (LHPs); thermoelectric coolers
- Thermal analysis tools: Thermal Desktop / ESATAN / custom FEM (Phase 10 applies)

#### Structures and Mechanisms
- Launch loads: vibration, acoustic, shock; structural design to launch environment (Phase 10)
- Lightweight materials: CFRP, Al-alloy 7075, titanium; specific stiffness selection
- Mechanisms: separation systems, solar array deployment, antenna deployment; release mechanisms

#### Command and Data Handling (CDH)
- On-board computer (OBC); memory (SRAM + NVRAM with ECC); data bus (MIL-STD-1553, SpaceWire, CAN)
- Telecommand and telemetry (TC/TM): uplink commands, downlink housekeeping data
- Fault detection, isolation, and recovery (FDIR): autonomy when communication is impossible

### 8. Model-Based Systems Engineering (MBSE)

- Traditional document-based SE: word documents → ambiguous, inconsistent, unlinked
- MBSE: single system model captures requirements, architecture, behaviour, parameters
- SysML: Systems Modeling Language; diagrams: Block Definition Diagram (BDD), Internal Block Diagram (IBD), Use Case, Activity, Sequence, State Machine, Requirement, Parametric
- Tools: Cameo Systems Modeler, OpenMBEE, Capella (free), SysML v2 (emerging standard)

## Completion Criteria

You have mastered this phase when you can:
1. Write a requirements document for a complex system, decompose it to subsystem level, and assign verification methods
2. Construct an FMEA for a critical subsystem and identify all single-point failures
3. Compute system reliability from a reliability block diagram with mixed series/parallel paths
4. Balance a spacecraft power budget over an orbit including eclipse
5. Describe the V-model and explain what happens at each phase; distinguish verification from validation
